The Brain —
Detection Engine

A three-layer cascade that classifies any text input as scam or legitimate in under 500ms. Designed to run on CPU with no GPU required, consuming minimal resources while maintaining enterprise-grade accuracy.

🧠
What is the Brain?
A plain-English explanation — no technical background required.

Imagine three security guards standing at a gate, in order. The first guard is very fast and knows exactly what a scammer looks like from a checklist of known tricks. If they spot something obvious — they stop it immediately.

If the first guard is unsure, the second guard is a statistician. They have studied thousands of past scam messages and can spot hidden patterns — things like "this message has too many urgency words and a suspicious-looking link."

If the second guard is still not sure, the third guard is a language expert. They actually read and understand the meaning of the message — catching sophisticated scams that cleverly avoid trigger words.

Key insight: ~60% of scam traffic is caught by Guard 1 in ~1ms. Only the hardest ~15% of cases ever reach Guard 3. This keeps the system fast and cheap.
What the Brain can analyze
💬 SMS / Chat messages 📧 Emails (subject + body) 🔗 URLs and links 📋 Any raw text
🏗
How the Cascade Works
The three-layer funnel — each layer only runs if the previous one is uncertain.
1
Rule Engine ~1ms
Runs 25 compiled regular expressions against the input. Catches known scam templates instantly: OTP harvesting, fake bank alerts, government impersonation, prize scams, malicious IP-based URLs.
✓ Exits here if score > 88% or < 8%
2
XGBoost ML ~10ms
Extracts 28 numerical features (word counts, ratios, URL signals, domain entropy, brand mentions) and feeds them to a trained gradient-boosted tree model. SHAP values identify the top contributing factors.
✓ Exits here if ML confidence > 82% or < 15%
3
TinyBERT NLP ~200ms
Fine-tuned transformer model that understands the meaning of the message. Catches social engineering that deliberately avoids all keywords — e.g. professionally worded Business Email Compromise attacks.
← Only ~15% of traffic reaches here
Ensemble Weighting Final Score Formula 
# Weights must sum to 1.0
Final = 0.20 × rule_score
       + 0.35 × xgboost_score
       + 0.45 × tinybert_score

# Then scale to 0–100
RiskScore = int(Final × 100)

Layer 1 — Rule Engine
rule_engine.py — the fastest, cheapest first check.

The Rule Engine is a list of 25 regular expressions — patterns that describe what scam text looks like. Think of it like a spam filter but smarter, with each pattern having a weight reflecting how strongly it indicates a scam.

Pattern Categories
● Urgency pressure language
● Account/credential harvesting
● Financial lures & prizes
● Bank / government impersonation
● Suspicious URL patterns
● Social engineering phrases

The Noisy-OR formula combines multiple pattern weights into a single score:

# If pattern A (weight 0.7) AND pattern B (weight 0.8) both match:
combined = 1 - (1 - 0.7) × (1 - 0.8)
         = 1 - 0.3 × 0.2
         = 1 - 0.06 = 0.94  ← very high = definite scam

# Why not just add the weights?
# Addition can exceed 1.0. Noisy-OR always stays in [0, 1].
Usage Python 
from brain.rule_engine import score

result = score("Your Maybank account is suspended. Verify OTP now!")

print(result.score)       # 0.97
print(result.matched)     # ['impersonate_bank', 'credential_suspend', 'credential_otp']
print(result.is_definite) # True  → cascade exits here, no ML needed

Layer 2 — XGBoost ML
ml_model.py — statistical pattern learning from thousands of examples.

XGBoost builds a collection of 300 decision trees, where each tree learns from the mistakes of the previous one. This "boosting" approach is why it's so accurate even on structured numerical data.

Training — what happens step by step
  1. Split data: 85% training, 15% validation
  2. Build Tree 1: tries to classify all samples
  3. Identify mistakes from Tree 1
  4. Build Tree 2: focuses on the mistakes of Tree 1
  5. Repeat 300 times
  6. Early stopping: halt if no improvement for 30 rounds (prevents overfitting)
  7. Save final ensemble to models/xgb_model.json
from brain.ml_model import train_from_dataframe, MLModel
import pandas as pd

# Train
df = pd.read_csv("data/sms_spam.csv")
model = train_from_dataframe(df, text_column="text", label_column="label")

# Or load previously trained
model = MLModel.load()

# Predict (needs feature vector from features.py)
from brain.features import extract
fv = extract("Click here to claim your free gift NOW")
result = model.predict(fv.to_list())

print(result.score)       # 0.89
print(result.explanation) # ['urgency_word_count', 'financial_word_count', 'has_short_url']

Layer 3 — TinyBERT NLP
nlp_model.py — semantic understanding of language meaning.

TinyBERT was pre-trained on Wikipedia and books — it understands English grammar and meaning. Fine-tuning is the process of nudging those weights using your scam/ham examples, teaching it what scam language looks like beyond keywords.

Why this catches what the others miss

Consider this sophisticated BEC (Business Email Compromise) message:

"Hi Sarah, as discussed in our call, please proceed with the payment of RM 47,000 to the new supplier account. I'm in a meeting and can't call — just reply to confirm and I'll approve."

⚠ Rule engine: 0.12 (no obvious patterns)
⚠ XGBoost: 0.44 (borderline)
✗ TinyBERT: 0.87 — understands the social engineering context 

# Fine-tune (run once, takes ~20-40min on CPU)
from brain.nlp_model import fine_tune
import pandas as pd

df = pd.read_csv("data/sms_spam.csv")
model = fine_tune(df, epochs=3, batch_size=16)

# Load and predict
from brain.nlp_model import NLPModel
model = NLPModel.load()
result = model.predict("Please transfer the funds immediately")
print(result.score)  # 0.81

Feature Engineering
features.py — 28 numerical signals extracted from every text input.

Features are the numbers that the XGBoost model actually sees. Instead of raw text, it gets 28 measurements — think of it like a blood test panel where each number tells the doctor something specific.

📝 Lexical (7 features)
char_count word_count digit_ratio caps_ratio text_entropy avg_word_length special_char_ratio
⚡ Urgency (3 features)
urgency_word_count urgency_ratio exclamation_count
💰 Financial (3 features)
financial_word_count has_currency_symbol has_amount_pattern
🔗 Link / URL (8 features)
url_count has_ip_url has_short_url has_suspicious_tld max_url_length max_domain_entropy has_login_path has_http_only
🏷 Identity (2 features)
brand_mention_count has_generic_greeting
🏗 Structure (5 features)
question_mark_count line_count has_phone_number has_email_address

Pipeline Orchestrator
pipeline.py — the Brain class that ties everything together.

This is the only class the rest of the system (API, UI) ever needs to touch.

from brain import Brain

# Load all models (call once at startup)
brain = Brain.load()

# Analyze any text
result = brain.analyze("Your CIMB account is suspended. Verify at http://1.2.3.4/login")

print(result.score)         # 94
print(result.verdict)       # "scam"
print(result.explanation)   # "HIGH CONFIDENCE SCAM. Top signals: Bank impersonation, IP-based URL, Account suspension threat."
print(result.layers_used)   # ["rule"]  — caught at layer 1, done in 1ms
print(result.latency_ms)    # 1.3

# Analyze email
result = brain.analyze_email(
    subject  = "Urgent: Action Required",
    body     = "Dear customer, please verify...",
    sender   = "security@maybank-verify.xyz",
)

# Get full dict (for API response)
print(result.to_dict())

Training Guide
How to train the models from scratch — step by step.
Step 1 — Prepare your data data/

Download these free datasets and place in the data/ folder:

  1. sms_spam.csv — SMS Spam Collection (Kaggle) — columns: text, label
  2. enron_email.csv — Enron Email Dataset (Kaggle) — columns: text, label
  3. phishtank.csv — PhishTank CSV dump (phishtank.com) — columns: url, label
Step 2 — Install dependencies 
pip install xgboost scikit-learn shap pandas numpy
pip install torch transformers   # for TinyBERT (optional but recommended)
Step 3 — Run training ~30-60 min 
# From the project root:
python -m brain.training

# Skip TinyBERT if you want faster training (~5 min):
python -m brain.training --skip-nlp

# Or from Python:
from brain import train_all
train_all(data_dir="data/")
Expected output after training: Precision > 93%, Recall > 88%, F1 > 90%. If numbers are lower, try adding more training data or increasing nlp_epochs to 5.
